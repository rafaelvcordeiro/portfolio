Why I created this webapp?

This project was initially conceived as CS50's Introduction to Computer Science course final project. And I decided to build a movie bookmarker because it is based on an issue I frequently deal with: what is my next movie or tv series I want to watch? There are three main reasons that supported this choice:

1. Whenever a friend recommended a movie or tv show, I would write it somewhere, and by the time I was finally available to watch it, I just couldn't find it, wasting quite some time.

2. I realized that there are two main factors that guides the decision of what to watch next: the current mood of the viewers and the available time they have. This process of deciding often takes quite some time as well. That's why this webapp has filters for Genres and Duration to aid selecting faster and more accuratelly what is really needed.

3. I wanted a computer science project that would challenge myself and that I could showcase to others, as I am constantly improving my backend software engineering skills.

Tools and resources used:

Flask: the used backend framework is Flask, as it is simple to implement, reliable and light. I'm using a Python app that manages the routes and requests from the user, while also managing each user session.
SQLite3: used sqlite3 as the database, and found the following implementation guidelines referenced here and here. This webapp uses two SQLite tables: USERS, that holds users data and RECORDS, that holds all data from each user.
Cinemagoer API: to make it easy to add new entries, the Cinemagoer IMDB API was used, to fill in the add items forms and improve usability.
Skeleton: used Skeleton CSS framework to increase frontend development speed. The Skeleton project seems to be no longer active, but for the basic usage of this webapp where I wanted a minimalistic approach, it worked and suited my needs perfectly well.
Memegen: used the Memegen meme generator for writing automatic error messages and information.
